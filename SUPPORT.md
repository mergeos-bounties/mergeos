# Support

Use the right channel so maintainers can respond quickly.

## Bugs

Open a bug report with reproduction steps, expected behavior, actual behavior, environment details, and evidence.

## Feature Requests

Open a feature request with the user problem, proposed solution, and acceptance criteria.

## Bounty Claims

Use the bounty claim template before starting bounty work. Bounty PRs should link the accepted claim and follow [BOUNTY-POLICY.md](BOUNTY-POLICY.md).

## Security

Do not open public issues for suspected vulnerabilities. Follow [SECURITY.md](SECURITY.md) and use private vulnerability reporting:

https://github.com/mergeos-bounties/mergeos/security/advisories/new

## Production

Production service status is maintained by repository maintainers. Do not post credentials, tokens, customer data, server details, or private deployment logs in public issues.
